CuteHttpFileServer（chfs）| 让小白也会使用电脑局域网传文件

晓皎 2020-02-20 18:11:30 4,368 本文共2488个字，预计阅读需要7分钟。

前言

之前我写过一篇 移动端怎么用局域网传文件的文章，这次把电脑端的补上。

为了防止大家不知道这是干什么的，我再说说使用场景。

想象一下现在你需要把电脑上的一部 66 GB 4K 电影传到家里另一台电脑/手机，你会怎么做？

用 QQ/微信/网盘？你想了想，可能会口吐芬芳。

那用移动硬盘/U 盘？首先你得有一个大容量的移动硬盘/U 盘（我以前是用 32GB U 盘传的，因为 U 盘太小装不了，还急中生智，使用分卷压缩解决了）；其次用移动硬盘/U 盘传给手机怕是还需要 OTG 转换头吧。你会说，谁 TM 会往手机传一部 66 GB 4K 电影，打个比方而已嘛，别较真。

那用 Windows 系统自带的局域网传文件功能，怎么说？也不是不可以，就是对小白来说配置比较麻烦。

终极解决之道还是用局域网。市面上有很多传文件软件，然而不少软件都需要双方都安装他们的软件才能用，比如：飞秋、AirDroid。你想想你传一个文件给别人，还要别人也装个同样的软件，对方不得口吐芬芳？

本着不麻烦别人的方针，我们需要用一个软件就完成分享的任务。当然这个软件不仅能分享文件，还能接收别人传来的文件那就更好了。意思是双方仅需一人安装软件，即可实现双方的文件分享与接收，岂不美哉？

经过多方查找，查看了诸多此类文章，仅找到两款软件，具体请看下面的详细描述。

局域网传文件温馨提示：

局域网传文件只要双方在同一网络、WiFi、热点情况下，都可以。其传输速度取决于路由器、网卡、手机 WiFi 模块等，这就是千兆路由器显威力的时候了。

优先连接同一网络、WiFi，在上述二者都没有的情况下才开热点传文件（为了防止意外偷跑流量，传输时可将数据流量关闭），因为手机 WiFi 模块性能不强，速度不快，一般就几 MB/s。

CuteHttpFileServer（chfs）

这款由国人开发的开源软件 CuteHttpFileServer，简称 chfs，是我到目前为止用过最舒服的局域网软件。我们看名字就知道通过 HTTP 传输，即浏览器传输，正合我意。

CuteHttpFileServer（chfs）官网：http://iscute.cn/chfs

GitHub 项目主页：https://github.com/docblue/chfsgui

主要特性

  • 单文件，免安装
  • 支持分享、接收、删除
  • 在线预览部分视频
  • 查看图片缩略图
  • 支持设置账户访问权限读、写、删除
  • 支持设置页面公告栏

首先保证两台设备在同一网络下，第一步添加共享文件夹（支持拖动文件夹添加，暂不支持拖动单个文件添加），第二步开启局域网，其次在软件右上角会显示 192.168 开头的 IP 地址，最后在另一台电脑/手机的浏览器内输入 IP 地址，即可打开共享文件界面。

默认支持分享、接收（即读、写），不支持删除，可在设置中进行修改，亦可添加多个账户，设置不同权限。

如上图所示，作者还很贴心的给手机准备了扫码下载功能。

视频可以在线看（受编码等影响，并不是所有视频都能在线看），当然你也可以在手机上长按复制链接（电脑上鼠标右键复制链接），使用 PotPlayer（电脑）、MX Player（安卓）、nPlayer（iOS/安卓）、VLC（全平台）等视频播放器播放，实测 1080P 高码率视频不会卡顿，播放流畅。

图片可以在设置中设置是否开启预览缩略图，实测有些图片可以，有些不行，不知为何。

测试一下电脑与手机传输一个系统镜像文件的速度，如上图所示，由于设备原因仅能跑满百兆，但依旧很快了。如果你的路由器、网线、网卡、WiFi 模块等一系列设备都不错的话，跑个千兆，万兆，十万兆也没有问题。

关于 CuteHttpFileServer（chfs）的软件下载问题，上图已经说得很明白了。命令行程序一般人不易上手，我们下载 GUI 程序就好了。正是因为 CuteHttpFileServer（chfs）早期没有图形界面，我才没有推荐。

当然如果你要用命令行程序，可查看如下两篇文章：

  • 异次元软件世界：https://www.iplaysoft.com/chfs.html
  • 小众软件：https://meta.appinn.net/t/cutehttpfileserver-chfs-nas-http-webdav/13634/1

Http File Server（HFS）

Http File Server（HFS）官网：https://www.rejetto.com/hfs/

此软件虽然也很不错，但由于有上一款软件的存在，此软件仅作为备用选择。其与 CuteHttpFileServer（chfs）相比主要主要有以下几点不如：

  • 没有官方中文（有第三方的汉化版）
  • 不支持接收（可能我没有找到）
  • UI 界面不够现代

使用方法也很简单，和 CuteHttpFileServer（chfs）差不多。第一步直接拖动文件到左侧的框里，第二步在另一台电脑/手机的浏览器内输入 IP 地址，即可打开共享文件界面。

Your current file system is not saved. Save it?

Http File Server（HFS）退出时会弹出如上提示，意思是你当前上传的文件是否记录，下次打开软件就不用再添加文件了，一般人传过一次就不会传第二次，选否就行了。

其他的也不多 BB，如果需要汉化版本，或进阶使用技巧，我找到一篇现成的文章，我亦无需赘述。由于其网站不能通过一般方式访问，我保存了一份 PDF 文件供大家查阅（由于 PDF 无法查看动图，我又单独下载了动图），观看体验可能不是很好。
